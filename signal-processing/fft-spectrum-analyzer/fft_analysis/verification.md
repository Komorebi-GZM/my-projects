- 结果一致性: ✅ (|X_dft - X_fft| = 2.79e-14)

### 2.3 复杂度分析结论

#### DFT 复杂度: O(N²)

直接按 DFT 定义计算：

```
X[k] = Σ_{n=0}^{N-1} x[n] · exp(-2πi · k · n / N)
```

- 对每个频点 k（共 N 个），需要遍历所有时域点 n（共 N 个）
- 总运算量: N × N = N² 次复数乘加
- N=4096 时: ~16.8 百万次运算

#### FFT 复杂度: O(N log N)

Cooley-Tukey 基2 DIT 算法利用旋转因子的对称性和周期性：

```
W_N^{k+N/2} = -W_N^k    (对称性)
W_N^{k+N}    =  W_N^k    (周期性)
```

- 递归分解: log₂(N) 层
- 每层执行 N/2 次蝶形运算
- 每个蝶形运算: 1 次复数乘法 + 2 次复数加减
- 总运算量: (N/2) · log₂(N) 次蝶形 ≈ O(N log N)
- N=4096 时: ~24,576 次蝶形运算

| N | DFT (N²) | FFT (N log₂ N) | 加速比 |
|---|----------|----------------|--------|
| 64 | 4,096 | 384 | 10.7x |
| 256 | 65,536 | 2,048 | 32.0x |
| 1,024 | 1,048,576 | 10,240 | 102.4x |
| 4,096 | 16,777,216 | 49,152 | 341.3x |
| 16,384 | 268,435,456 | 229,376 | 1,170.0x |
| 65,536 | 4.3 × 10⁹ | 1,048,576 | 4,096.0x |

> 注意: 上表为理论运算量对比。实测 np.fft.fft 使用高度优化的 C/Fortran 实现，
> 而 my_fft 为纯 Python 递归实现，实测时间差异远大于理论运算量差异。
> 但复杂度增长趋势一致（参见 benchmark.png 中复杂度趋势图）。
