# ADR-003: 本地优先开发工作流

## Status
Accepted

## Date
2026-05-10

## Context

项目采用CloudBase云开发作为生产环境，面临开发工作流的选择：

**选项1：云优先** - 直接在CloudBase上开发和测试
- 优点：开发与生产环境一致
- 缺点：需要网络连接，调试困难，测试数据污染风险

**选项2：本地优先** - 本地完整开发，统一部署
- 优点：开发体验好，测试快速，离线可工作
- 缺点：需要维护两套环境的一致性

## Decision

采用**本地优先开发工作流**，所有业务逻辑在本地（Flask + SQLite）完整开发和测试，通过适配层保证与生产环境（CloudBase）兼容。

### 工作流架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        开发阶段                                  │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐      │
│  │   本地开发    │ → │   本地测试    │ → │  本地联调    │      │
│  │  Flask+SQLite│    │    pytest    │    │ 小程序+本地API│      │
│  └──────────────┘    └──────────────┘    └──────────────┘      │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                        部署阶段                                  │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐      │
│  │   数据迁移    │ → │  云函数部署   │ → │  小程序发布  │      │
│  │ SQLite→NoSQL │    │ Flask→云函数  │    │  提交审核    │      │
│  └──────────────┘    └──────────────┘    └──────────────┘      │
└─────────────────────────────────────────────────────────────────┘
```

### 开发环境配置

```python
# src/config.py
class DevelopmentConfig(Config):
    DEBUG = True
    DB_MODE = "local"  # 使用SQLite

class ProductionConfig(Config):
    DB_MODE = "production"  # 使用CloudBase NoSQL
```

### 本地开发优势

1. **快速反馈**：pytest测试秒级完成
2. **离线开发**：无需网络连接
3. **调试友好**：IDE断点、日志查看
4. **数据可控**：SQLite文件可随时重置
5. **成本为零**：无需云资源费用

### 部署流程

```bash
# 1. 本地开发完成，测试通过
pytest tests/ -v

# 2. 预览部署
python scripts/full_deploy.py --env-id <env-id> --dry-run

# 3. 执行部署
python scripts/full_deploy.py --env-id <env-id>

# 4. 小程序上传发布
# 使用微信开发者工具上传代码
```

## Alternatives Considered

### 方案A：云开发优先
- **Pros**: 环境与生产完全一致
- **Cons**: 每次修改需上传部署，调试困难，测试慢
- **Rejected**: 开发效率太低

### 方案B：容器化本地环境（Docker）
- **Pros**: 环境一致性更好
- **Cons**: 增加复杂度，CloudBase难以本地模拟
- **Rejected**: 过度设计，当前方案足够

### 方案C：使用CloudBase本地模拟器
- **Pros**: 官方支持，与生产接近
- **Cons**: 功能有限，不稳定
- **Rejected**: 成熟度不足

## Consequences

### Positive
- 开发体验接近传统Web开发
- 测试速度快，反馈及时
- 新成员上手成本低
- 不依赖云服务可用性

### Negative
- 需要维护本地/生产两套配置
- 某些云特定功能无法在本地测试
- 部署前才能发现环境差异问题

### Mitigations
- 完善的部署前检查清单
- 自动化部署脚本减少人为错误
- 集成测试覆盖主要流程
- 预发布环境验证（如需要）
