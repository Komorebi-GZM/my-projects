## 二、CloudBase 环境配置

### 2.1 环境创建

1. 登录微信开发者工具
2. 点击"云开发" → "创建环境"
3. 选择"免费基础版"
4. 记录环境ID（如：`campus-storage-xxx`）

### 2.2 数据库集合创建

在 CloudBase 控制台创建以下集合：

| 集合名 | 说明 | 索引 |
|--------|------|------|
| users | 用户表 | openid (唯一) |
| orders | 订单表 | orderNo (唯一), openid, status |
| warehouses | 仓库表 | - |
| tickets | 配送单表 | orderId |

### 2.3 环境变量配置

在云函数配置中设置以下环境变量：

```bash
# 基础配置
TCB_ENV_ID=your-env-id

# 微信小程序
WX_APPID=your-app-id
WX_SECRET=your-app-secret

# 管理员
ADMIN_OPENIDS=openid1,openid2

# 业务参数
STORAGE_FEE_RATE=0.5
TIMEOUT_DAYS=7
DELIVERING_TIMEOUT_HOURS=24
```

## 三、部署执行步骤

### 步骤1：数据迁移

```bash
# 预览模式（先验证）
python scripts/migrate_to_cloudbase.py --env-id <环境ID> --dry-run

# 实际迁移
python scripts/migrate_to_cloudbase.py --env-id <环境ID>

# 验证迁移结果
python scripts/verify_migration.py --env-id <环境ID>
```

### 步骤2：云函数部署

```bash
# 预览模式
python scripts/deploy_cloudfunctions.py --env-id <环境ID> --dry-run

# 实际部署
python scripts/deploy_cloudfunctions.py --env-id <环境ID>
```

部署顺序：
1. login
2. createOrder
3. getOrderList / getOrderDetail
4. updateOrderStatus
5. pendingTimeoutChecker / deliveringTimeoutChecker

### 步骤3：定时触发器配置

在 CloudBase 控制台配置定时触发器：

| 云函数 | Cron表达式 | 说明 |
|--------|-----------|------|
| pendingTimeoutChecker | `0 */30 * * * * *` | 每30分钟执行 |
| deliveringTimeoutChecker | `0 */30 * * * * *` | 每30分钟执行 |

### 步骤4：小程序配置更新

修改 `miniprogram/utils/request.js`：

```javascript
// 生产环境
const API_BASE_URL = 'https://<环境ID>.service.tcloudbase.com';
```

### 步骤5：小程序上传

1. 微信开发者工具 → 点击"上传"
2. 填写版本号（如：`1.0.0`）
3. 填写项目备注
4. 点击"上传"

### 步骤6：提交审核

1. 登录微信公众平台
2. 进入"版本管理"
3. 找到开发版本 → 点击"提交审核"
4. 填写审核信息
5. 等待审核通过

### 步骤7：正式发布

审核通过后：
1. 进入"版本管理"
2. 点击"发布"
3. 确认发布
