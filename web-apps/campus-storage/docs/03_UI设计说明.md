# 校园物品暂存平台 · UI设计说明

> 版本：v1.0
> 创建日期：2026-05-10
> 最后更新：2026-05-10
> 状态：草稿

---

## 1. 设计原则

1. **极简操作**：目标用户是学生，流程尽量简化，3步内完成核心操作
2. **信息透明**：费用、状态、位置清晰可见，建立信任感
3. **容错设计**：支持修改地址、取消订单、异常状态友好提示

---

## 3. 用户端页面设计

### 3.1 首页

**页面结构**：顶部Banner轮播 → 快捷操作区 → 进行中订单卡片 → 底部TabBar

**核心元素**：

- **Banner轮播区**：展示平台活动、使用引导等内容，自动轮播，支持手动滑动
- **快捷操作区**：两个等宽按钮卡片——「寄存物品」（主色填充，白色图标+文字）和「我的订单」（白色底色，主色边框+文字）
- **进行中订单卡片**：展示当前进行中的订单摘要信息，包含物品类型、当前状态、状态颜色标识；若无进行中订单则显示空状态引导
- **底部TabBar**：三个Tab——「首页」「订单」「我的」

**交互逻辑**：

- 点击「寄存物品」跳转至下单页
- 点击「我的订单」跳转至订单列表页
- 点击进行中订单卡片跳转至对应订单详情页
- TabBar切换实现页面间导航
- Banner点击可跳转至对应活动页或功能页

---

### 3.2 下单页

**页面结构**：选择暂存城市 → 填写物品信息 → 选择服务方式 → 费用预览 → 提交订单

**核心元素**：

- **城市选择器**：点击弹出城市列表弹窗，支持搜索筛选，选中后回填显示
- **物品信息表单**：
  - 物品类型：下拉选择（行李箱、书籍、电子产品、衣物、其他）
  - 物品体积：下拉选择（小件/中件/大件），附带尺寸参考说明
  - 物品重量：输入框（kg），支持小数点后一位
  - 备注说明：多行文本输入框（选填），placeholder提示"请描述物品特殊情况"
- **服务方式选择**：两个单选项卡片——「上门取件」（含取件地址输入）和「自送仓库」（含仓库地址及地图展示）
- **费用预览卡片**：根据已填信息实时计算，展示暂存费用、取件费用（如有）、合计金额
- **提交订单按钮**：底部固定，主色填充，校验通过后可点击

**交互逻辑**：

- 表单字段实时校验，未填必填项时提交按钮置灰不可点击
- 切换城市或物品信息时费用预览实时更新
- 选择「上门取件」时展开取件地址填写区域，选择「自送仓库」时折叠
- 提交时弹出确认弹窗，展示订单摘要信息，确认后创建订单
- 提交成功后跳转至订单详情页，同时展示成功Toast提示

---

### 3.3 订单列表页

**页面结构**：Tab切换栏 → 订单卡片列表 → 空状态占位

**核心元素**：

- **Tab切换栏**：四个Tab——「全部」「进行中」「已完成」「已取消」，当前选中Tab底部主色下划线高亮
- **订单卡片**：每张卡片包含以下信息
  - 顶部：订单编号 + 状态标签（对应状态颜色）
  - 中部：物品类型图标 + 物品描述 + 暂存城市
  - 底部：下单时间 + 操作按钮（根据状态动态显示，如「查看详情」「去支付」「再次下单」）
- **空状态**：插图 + 文案「暂无订单，去寄存物品吧」+ 引导按钮

**交互逻辑**：

- Tab切换时列表数据筛选刷新，切换过程使用骨架屏过渡
- 下拉刷新获取最新订单列表
- 上拉加载更多（分页加载）
- 点击订单卡片整体区域跳转至订单详情页
- 首次进入默认展示「全部」Tab

---

### 3.4 订单详情页

**页面结构**：状态时间轴 → 物品信息卡片 → 费用信息卡片 → 操作按钮区

**核心元素**：

- **状态时间轴**（纵向步骤条）：
  - 每个节点包含：状态图标、状态文案、时间戳
  - 已完成节点为绿色实心圆点，当前节点为蓝色脉冲动画，未到达节点为灰色空心圆点
  - 节点之间用竖线连接，已完成段为绿色，未完成段为灰色
- **物品信息卡片**：物品类型、体积、重量、暂存城市、服务方式、下单时间
- **费用信息卡片**：暂存费用、取件费用、配送费用（如有）、改址费用（如有）、合计金额
- **操作按钮区**（底部固定，根据订单状态动态显示）：
  - PENDING状态：「取消订单」
  - STORED状态：「发起配送」「调整目的地」
  - DELIVERING状态：「确认签收」
  - COMPLETED状态：「再次下单」
  - EXCEPTION状态：「联系客服」
  - CANCELLED状态：无操作按钮

**交互逻辑**：

- 进入页面时默认展示当前状态对应的时间轴位置
- 操作按钮点击后弹出二次确认弹窗
- 取消订单需填写取消原因
- 发起配送跳转至配送发起页，调整目的地跳转至目的地调整页

---

### 3.5 配送发起页

**页面结构**：填写配送地址 → 选择配送时间 → 费用预览 → 确认支付

**核心元素**：

- **配送地址表单**：
  - 收件人姓名：文本输入框
  - 联系电话：数字输入框，限制11位手机号
  - 配送地址：文本输入框，支持调用微信地址选择接口
- **配送时间选择**：日期选择器 + 时段选择（上午9-12点/下午12-18点/晚上18-21点），不可选过去时间
- **费用预览卡片**：配送费用明细 + 暂存费用合计 + 本次支付金额
- **确认支付按钮**：底部固定，主色填充

**交互逻辑**：

- 地址信息实时校验，手机号格式校验
- 选择配送时间后费用预览实时更新
- 点击确认支付弹出支付确认弹窗，确认后调用微信支付
- 支付成功跳转至订单详情页，状态更新为DELIVERING
- 支付失败展示错误Toast提示

---

### 3.6 目的地调整页

**页面结构**：当前信息展示 → 选择新城市 → 改址费用说明 → 确认支付

**核心元素**：

- **当前信息展示卡片**：当前暂存城市、物品信息摘要，灰色底色表示不可编辑
- **新城市选择器**：点击弹出城市列表弹窗，与下单页城市选择器交互一致
- **改址费用说明卡片**：
  - 原暂存费用
  - 新暂存费用
  - 改址手续费（固定金额或按规则计算）
  - 差价说明（多退少补）
- **确认支付按钮**：底部固定，需补差价时显示支付金额，有退款时显示「确认改址（退还XX元）」

**交互逻辑**：

- 新城市不可与当前城市相同，相同时给出提示
- 选择新城市后费用说明实时更新
- 确认后弹出二次确认弹窗，展示新旧城市对比信息
- 确认支付后订单状态保持STORED，目的地信息更新
- 支付/退款流程与配送发起页一致

---

### 3.7 个人中心页

**页面结构**：用户头像+昵称区 → 功能入口列表 → 关于我们

**核心元素**：

- **用户信息区**：圆形头像（支持点击更换）+ 昵称 + 微信授权手机号（脱敏显示）
- **功能入口列表**（Cell组件列表）：
  - 我的订单（带进行中订单数量角标）
  - 投诉/报损（跳转投诉提交页）
  - 联系客服（拨打电话或跳转客服会话）
  - 关于我们（跳转关于页面）
- **关于我们**：底部展示版本号信息

**交互逻辑**：

- 首次进入引导微信授权登录，获取头像和昵称
- 点击「我的订单」跳转至订单列表页
- 点击「投诉/报损」跳转至投诉提交页，需选择关联订单
- 点击「联系客服」调用 `wx.makePhoneCall` 拨打客服热线

## 4. 运营端页面设计

### 4.1 订单管理页

**页面结构**：搜索栏 + 筛选条件 → 订单列表 → 批量操作栏

**核心元素**：

- **搜索栏**：支持按订单号、用户手机号搜索
- **筛选条件**：状态下拉筛选、日期范围选择、城市筛选
- **订单列表**：紧凑型卡片列表，展示订单号、物品类型、状态、用户信息、下单时间
- **批量操作栏**：底部固定，支持全选/批量导出功能

**交互逻辑**：

- 搜索支持输入防抖，300ms后触发搜索
- 筛选条件变更后列表实时刷新
- 点击订单卡片跳转至运营端订单详情
- 支持长按进入多选模式，底部弹出批量操作栏

---

### 4.2 入库操作页

**页面结构**：扫码/手动输入区 → 物品信息确认 → 拍照上传 → 确认入库

**核心元素**：

- **扫码/手动输入区**：扫码按钮（调用摄像头扫码）+ 手动输入订单号文本框
- **物品信息确认卡片**：扫码/输入后展示关联订单的物品信息，供工作人员核对
- **拍照上传区**：调用相机拍照，支持多张照片上传（物品外观、入库标签等）
- **确认入库按钮**：主色填充，点击后订单状态更新为STORED

**交互逻辑**：

- 扫码成功自动填充订单号并查询关联信息
- 手动输入支持回车搜索
- 订单不存在或状态不符时展示错误提示
- 拍照上传支持预览和删除已拍照片
- 确认入库后展示成功提示，自动清空当前表单，准备下一个入库操作

---

### 4.3 异常处理页

**页面结构**：异常订单列表 → 异常详情 → 处理方案选择

**核心元素**：

- **异常订单列表**：卡片列表，展示订单号、异常原因标签、上报时间、处理状态（待处理/处理中/已结案）
- **异常详情弹窗/页**：展示订单完整信息 + 异常原因描述 + 现场照片（如有）
- **处理方案选择**：
  - 标记异常原因：下拉选择（物品损坏、物品丢失、信息不符、其他）+ 补充说明文本框
  - 处理方案：两个操作按钮——「结案」（标记为已处理，订单继续流转）和「作废」（订单终止，触发退款流程）

**交互逻辑**：

- 异常订单列表默认按上报时间倒序排列
- 点击异常订单进入详情处理页
- 选择处理方案后弹出二次确认弹窗
- 作废操作需额外填写作废原因
- 处理完成后订单状态更新，列表自动刷新

## 6. 交互细节

### 6.1 按钮状态

| 状态 | 样式 |
|------|------|
| 可点击 | 主色（`#1890FF`）填充，白色文字 |
| 不可点击 | 灰色（`#CCCCCC`）填充，灰色文字，禁用图标 |
| 按压中 | 主色加深（`#096DD9`），缩放0.98 |
| 加载中 | 按钮内显示loading图标，文字变为"提交中..."，不可重复点击 |

### 6.2 加载状态

| 场景 | 方案 |
|------|------|
| 订单列表页首次加载 | 骨架屏（3张卡片骨架） |
| 提交操作（下单/支付/入库等） | 按钮loading状态 |
| 页面切换 | 无额外loading，数据缓存+异步刷新 |

### 6.3 空状态

- 统一使用插图 + 文案 + 操作引导按钮的组合形式
- 示例：
  - 订单列表为空：插图 + "暂无订单，去寄存物品吧" + 「去寄存」按钮
  - 搜索无结果：插图 + "未找到相关订单" + 「清空筛选」按钮

### 6.4 错误提示

- 使用页面顶部 Toast 提示，2秒自动消失
- 网络异常：Toast文案"网络异常，请稍后重试"
- 表单校验失败：对应输入框下方红色提示文案
- 接口异常：Toast文案"操作失败，请稍后重试"

## 7. 页面跳转关系

```
首页 ──→ 下单页
     ──→ 订单列表页 ──→ 订单详情页 ──→ 配送发起页
     ──→ 个人中心页                    ──→ 目的地调整页
                                      ──→ 投诉页
```

**跳转方式说明**：

| 起始页 | 目标页 | 跳转方式 | 说明 |
|--------|--------|---------|------|
| 首页 | 下单页 | `wx.navigateTo` | 保留首页栈 |
| 首页 | 订单列表页 | `wx.switchTab` | TabBar页面切换 |
| 首页 | 个人中心页 | `wx.switchTab` | TabBar页面切换 |
| 订单列表页 | 订单详情页 | `wx.navigateTo` | 保留列表页栈 |
| 订单详情页 | 配送发起页 | `wx.navigateTo` | 保留详情页栈 |
| 订单详情页 | 目的地调整页 | `wx.navigateTo` | 保留详情页栈 |
| 订单详情页 | 投诉页 | `wx.navigateTo` | 保留详情页栈 |
