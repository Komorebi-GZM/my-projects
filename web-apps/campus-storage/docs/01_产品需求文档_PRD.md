## 一、文档概述

### 1.1 文档目的与范围

本文档是「校园物品暂存平台」Demo 版本的产品需求定义，适用于课程设计、毕业设计展示场景。文档覆盖用户端核心功能需求、非功能需求、业务流程及数据需求，为后续系统设计与开发提供依据。

**范围说明：**

- 本版本聚焦校园场景，目标用户为应届毕业生和跨校区学生
- 技术栈限定为微信小程序 + 腾讯 CloudBase 云开发（Serverless），后端云函数采用 Python 3.12 运行时
- Demo 版本不涉及真实支付对接、真实物流合作，采用模拟数据与手动流程

### 1.2 术语定义

| 术语 | 定义 |
|------|------|
| 暂存 | 用户将物品寄存在平台合作仓库，等待后续配送 |
| 配送 | 用户确定地址后，平台将物品从仓库送达指定地址 |
| 目的地调整 | 用户修改物品的暂存城市（核心差异化功能） |
| 状态机 | 订单从创建到完结的状态流转规则 |
| CloudBase | 腾讯云开发的 Serverless 云服务，提供云函数、云数据库、云存储等能力 |

## 二、产品概述

### 2.1 产品定位

```
FOR 校园毕业生和跨校区学生
WHO 需要跨校区/跨城市携带物品但目的地地址不确定
THE "校园行囊管家" IS A 物品暂存与配送平台
THAT 让用户提前寄存物品到目的地附近，确定地址后一键配送
UNLIKE 传统物流需要确定收货地址
OUR PRODUCT 支持"地址未定，物品先行"的创新场景
```

### 2.2 目标用户画像

**画像一：应届毕业生 - 小李**

```
基本信息：22岁，计算机专业应届生，正在求职
场景特点：毕业离校需搬走行李，但求职城市租房地址未定
核心痛点：
  - 毕业离校必须搬走宿舍物品，但新城市租房还没签
  - 行李寄给同学不方便，迷你仓价格贵还要自己搬
  - 面试多个城市，行李不知道该寄到哪
期望：把行李寄到求职城市附近暂存，确定租房后配送上门
```

**画像二：跨校区学生 - 小王**

```
基本信息：20岁，大二学生，本学期从老校区搬到新校区
场景特点：跨校区搬迁，新宿舍地址尚未分配
核心痛点：
  - 老校区宿舍要清退，新校区宿舍还没分好
  - 书籍、衣物等物品量大，自行搬运不方便
  - 临时寄存在同学宿舍占地方，不好意思长期放
期望：把物品暂存在新校区附近，分到宿舍后直接配送过去
```

### 2.3 核心价值主张

> **"地址未定，物品先行"**

解决校园场景中「必须搬走但没有确定目的地」的物品寄存难题，提供暂存 + 配送的一站式服务。

## 三、功能需求

### 3.1 用户故事清单

**US-001: 用户注册登录**

```
As a 新用户
I want to 通过微信快速登录
So that 我可以开始使用暂存服务

Acceptance Criteria:
- [ ] 支持微信一键授权登录
- [ ] 首次登录自动创建账户，获取用户昵称和头像
- [ ] 登录后跳转首页
```

**US-002: 物品寄存下单**

```
As a 已登录用户
I want to 发起物品寄存订单
So that 我的物品可以提前运到目的地附近暂存

Acceptance Criteria:
- [ ] 可选择暂存城市/校区（下拉选择）
- [ ] 可填写物品类型（书籍/衣物/电子产品/生活用品/其他）
- [ ] 可预估物品体积（小/中/大/超大）
- [ ] 可选择服务方式（上门取件/自送到仓）
- [ ] 可选择暂存天数（7天/15天/30天）
- [ ] 显示预估费用
- [ ] 支持提交订单（Demo阶段模拟支付）
```

**US-003: 物品状态查询**

```
As a 已下单用户
I want to 查看我的物品当前状态
So that 我随时知道物品在哪里

Acceptance Criteria:
- [ ] 显示订单列表（按时间倒序）
- [ ] 每个订单显示当前状态（待取件/运输中/已入库/待配送/配送中/已完成）
- [ ] 显示物品位置（城市+仓库名称）
- [ ] 显示剩余暂存天数
- [ ] 支持下拉刷新
```

**US-004: 发起配送**

```
As a 物品已入库的用户
I want to 发起配送请求
So that 物品可以送到我指定的地址

Acceptance Criteria:
- [ ] 可输入配送地址（支持校区宿舍楼/校外地址）
- [ ] 可选择配送时间（工作日/周末/指定日期）
- [ ] 显示配送费用
- [ ] 支持提交配送请求（Demo阶段模拟支付）
- [ ] 提交后生成配送单，订单状态更新
```

**US-005: 保价服务**

```
As a 寄存贵重物品的用户
I want to 为物品购买保价
So that 物品丢失或损坏时可以获得赔偿

Acceptance Criteria:
- [ ] 下单时可选择是否保价
- [ ] 可输入物品声明价值
- [ ] 显示保价费用（声明价值的1-3%）
- [ ] 保价信息显示在订单详情中
```

**US-006: 目的地调整**

```
As a 已下单但计划变更的用户
I want to 修改物品的暂存城市/校区
So that 我的物品可以跟随我的实际目的地

Acceptance Criteria:
- [ ] 运输中状态可发起改址请求
- [ ] 已入库状态可发起改址请求
- [ ] 可选择新的暂存城市/校区
- [ ] 系统自动计算改址费用（跨城转运成本）
- [ ] 显示改址后预计到达时间
- [ ] 确认改址费后执行转运
- [ ] 转运过程中实时更新状态
- [ ] 改址成功后通知用户
```

**US-007: 订单管理**

```
As a 已注册用户
I want to 管理我的所有订单
So that 我可以方便地查看和处理历史订单

Acceptance Criteria:
- [ ] 支持按状态筛选订单（全部/进行中/已完成/已取消）
- [ ] 支持查看订单详情（物品信息、费用明细、状态时间线）
- [ ] 支持取消待取件状态的订单
```

### 3.3 MVP 功能范围界定

Demo 版本仅实现 **P0 级别功能**，具体简化策略如下：

| 维度 | Demo 简化方案 |
|------|-------------|
| 地理范围 | 单城市（如：广州），支持 2-3 个模拟校区/仓库 |
| 登录方式 | 微信一键登录（CloudBase 内置 auth） |
| 支付流程 | 模拟支付（点击确认即视为支付成功） |
| 配送流程 | 模拟配送（管理员手动更新状态） |
| 仓库数据 | 预置模拟仓库数据，无需真实合作 |
| 消息通知 | Demo 阶段可暂不接入微信模板消息，使用站内消息 |
| 运营端 | 简单的管理页面（可嵌入同一小程序的管理入口） |

## 五、业务流程

### 5.1 用户端主流程

```
注册登录（微信授权）
    ↓
选择暂存城市/校区
    ↓
填写物品信息（类型、体积、描述）
    ↓
选择服务方式（上门取件 / 自送到仓）
    ↓
确认费用 → 模拟支付
    ↓
等待取件（PENDING）
    ↓
运输中（TRANSIT）→ 【可修改目的地】
    ↓
已入库（STORED）→ 【仍可修改目的地】
    ↓
发起配送 → 填写配送地址 → 确认配送费
    ↓
配送中（DELIVERING）
    ↓
确认收货（COMPLETED）
```

### 5.2 目的地调整子流程

```
用户发起改址请求
    ↓
系统判断物品当前状态：
  ├─ 运输中（TRANSIT）→ 拦截当前运输，重新发往新校区仓库
  ├─ 已入库（STORED）→ 从当前仓库调拨至新校区仓库
  └─ 其他状态 → 提示不可改址
    ↓
计算改址费用（跨校区/跨城转运成本）
    ↓
展示改址费用和预计到达时间
    ↓
用户确认支付改址费
    ↓
执行转运，更新订单状态
    ↓
通知用户改址成功及新预计到达时间
```

### 5.3 异常处理流程

```
管理员发现异常（物品损坏、地址无法送达、超时未取件等）
    ↓
标记订单为异常状态（EXCEPTION）
    ↓
人工处理：
  ├─ 联系用户协商解决方案
  ├─ 安排补发或赔偿
  └─ 记录处理过程
    ↓
处理完毕 → 结案（COMPLETED）或作废（CANCELLED）
```

## 六、数据需求

### 6.1 核心数据实体

| 实体 | 说明 | 关键属性 |
|------|------|---------|
| 用户 | 平台注册用户 | 微信 openid、昵称、头像、手机号（可选）、注册时间 |
| 订单 | 物品暂存/配送订单 | 订单号、状态、物品类型、体积、重量、暂存城市、配送地址、费用明细、是否保价、创建时间、更新时间 |
| 仓库 | 合作仓库信息 | 仓库名称、所属城市/校区、地址、容量、当前库存、联系人 |
| 操作日志 | 状态变更记录 | 订单号、变更前状态、变更后状态、操作人、操作时间、备注 |

### 6.2 数据库集合规划（CloudBase）

| 集合名 | 用途 | 权限 |
|--------|------|------|
| `users` | 用户信息 | 用户仅读写自己的记录 |
| `orders` | 订单数据 | 用户可读自己的订单，管理员可读写全部 |
| `warehouses` | 仓库信息 | 所有用户可读，仅管理员可写 |
| `operation_logs` | 操作日志 | 仅管理员可读写 |

## 八、附录

### 8.1 竞品分析摘要

| 竞品 | 服务模式 | 与本平台的差异 |
|------|---------|-------------|
| 菜鸟驿站 | 快递代收 + 短期暂存 | 需确定收货地址，不支持"地址未定"场景 |
