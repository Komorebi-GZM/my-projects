# Phase 6 前后端联调测试指南

## 概述

本指南描述如何在本地环境中进行前后端联调测试，包括微信小程序与后端API的联调步骤。

## 环境准备

### 1. 启动后端服务

```bash
# 进入项目目录
cd /sessions/6a00293006d386035f0448ed/workspace

# 启动Flask开发服务器
python -m src.app
```

后端服务将在 `http://localhost:5000` 启动。

### 2. 配置小程序开发环境

#### 安装微信开发者工具
- 下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- 安装稳定版

#### 导入小程序项目
1. 打开微信开发者工具
2. 点击「导入项目」
3. 选择 `/sessions/6a00293006d386035f0448ed/workspace/miniprogram` 目录
4. 填写 AppID（测试号或正式号）
5. 点击「导入」

### 3. 配置请求域名

由于微信小程序有域名白名单限制，本地开发需要以下配置：

#### 方法一：关闭域名校验（开发环境）
1. 在微信开发者工具中点击「详情」
2. 勾选「不校验合法域名、web-view（业务域名）、TLS版本以及HTTPS证书」

#### 方法二：配置合法域名
1. 登录微信公众平台
2. 进入「开发」→「开发管理」→「开发设置」
3. 在「服务器域名」中添加：
   - request合法域名：`http://localhost:5000`
   - 注意：生产环境必须使用HTTPS

## 联调测试步骤

### 测试1：用户登录

**测试目标**：验证小程序登录流程

**操作步骤**：
1. 在小程序首页下拉刷新
2. 观察控制台网络请求

**预期结果**：
- 请求：`POST http://localhost:5000/api/login`
- 响应：`{"code": 0, "data": {"openId": "...", "role": "USER"}}`
- 本地存储应保存用户信息

**调试检查点**：
- [ ] 请求头包含 `X-OpenId`
- [ ] 响应状态码 200
- [ ] 用户数据保存到 Storage

### 测试2：创建订单

**测试目标**：验证下单流程

**操作步骤**：
1. 点击首页「寄存物品」按钮
2. 选择城市、物品类型、仓库
3. 点击「立即下单」

**预期结果**：
- 请求：`POST http://localhost:5000/api/orders`
- 请求体：`{"city": "...", "itemType": "...", "warehouseId": "..."}`
- 响应：创建成功的订单数据
- 页面跳转到订单详情

**调试检查点**：
- [ ] 费用计算正确显示
- [ ] 提交按钮状态正确
- [ ] 下单成功后跳转详情页

### 测试3：订单列表

**测试目标**：验证订单列表加载

**操作步骤**：
1. 点击底部 TabBar 「订单」
2. 切换不同状态 Tab
3. 上拉加载更多

**预期结果**：
- 请求：`GET http://localhost:5000/api/orders?page=1&pageSize=10`
- 响应：订单列表数据
- 分页加载正常工作

**调试检查点**：
- [ ] Tab 切换触发重新加载
- [ ] 分页参数正确传递
- [ ] 空状态显示正确

### 测试4：订单详情

**测试目标**：验证订单详情和操作

**操作步骤**：
1. 点击任意订单卡片
2. 查看订单详情
3. 点击操作按钮（如「发起配送」）

**预期结果**：
- 请求：`GET http://localhost:5000/api/orders/{id}`
- 响应：完整订单数据
- 状态历史正确显示

**调试检查点**：
- [ ] 状态时间轴渲染正确
- [ ] 操作按钮根据状态显示
- [ ] 状态变更后页面刷新

### 测试5：状态变更

**测试目标**：验证订单状态流转

**操作步骤**：
1. 在订单详情页点击「发起配送」
2. 填写配送信息
3. 提交配送申请

**预期结果**：
- 请求：`PATCH http://localhost:5000/api/orders/{id}/status`
- 请求体：`{"newStatus": "DELIVERING", "operatorType": "USER"}`
- 响应：更新后的订单数据

**调试检查点**：
- [ ] 状态机校验正确工作
- [ ] 无效状态变更被拒绝
- [ ] 状态历史记录更新

## 常见问题排查

### 问题1：请求失败，提示 "fail url not in domain list"

**原因**：微信小程序域名白名单限制

**解决方案**：
1. 在微信开发者工具中关闭域名校验
2. 或配置 request 合法域名

### 问题2：后端返回 401 Unauthorized

**原因**：请求头缺少 X-OpenId

**解决方案**：
1. 检查 app.js 中登录逻辑是否正常
2. 检查 request.js 中请求头设置

### 问题3：CORS 跨域错误

**原因**：浏览器同源策略限制

**解决方案**：
1. 后端已配置 CORS，检查 `src/middleware/cors.py`
2. 确保请求头 `Content-Type: application/json`

### 问题4：数据不刷新

**原因**：页面缓存或状态管理问题

**解决方案**：
1. 使用 `onShow` 生命周期刷新数据
2. 检查数据绑定是否正确
