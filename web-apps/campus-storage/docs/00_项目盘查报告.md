**RICE计算公式**：`(Reach × Impact × Confidence) / Effort`

**评分标准**：

- Impact: Minimal=0.25, Low=0.5, Medium=1, High=2, Massive=3
- Confidence: Low=0.5, Medium=0.8, High=1
- Effort: S=1, M=2, L=3, XL=5（人月）

**Big Bets（高RICE + 高Effort）**：

### 3.2 开发模式说明

本项目采用**本地Python开发 → 统一部署**模式：

- **本地开发环境**：Flask + SQLite，在本地完成全部业务逻辑开发、单元测试和集成测试
- **生产部署环境**：CloudBase 云函数 + CloudBase NoSQL，通过 `db_adapter` 数据库适配层抹平差异
- **开发流程**：先在本地完整开发和测试，验证通过后统一部署上线至 CloudBase

### 3.3 路线图风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 跳过原型直接开发 | 可能导致大量返工 | 建议先完成低保真原型验证 |
| 未进行用户测试 | 可能开发出用户不需要的功能 | MVP发布后立即启动用户测试 |
| Demo阶段模拟数据 | 无法验证真实运营流程 | 明确Demo边界，规划生产环境路径 |

### 3.4 里程碑建议

```
Week 1-2: 用户访谈验证（5-8人）+ 价格敏感度测试
Week 3: 低保真原型设计 + 可用性测试
Week 4-6: MVP本地开发（P0功能，Flask + SQLite）
Week 7: 本地集成测试 + 内测（10-20人）

```

## 附录：RICE评分详细计算

### 计算公式

```
RICE Score = (Reach × Impact × Confidence) / Effort
```

### 评分标准详解

| 维度 | 分值 | 说明 |
|------|------|------|
| **Reach** | 0-100% | 在给定时间内会使用该功能的用户比例 |
| **Impact** | 0.25/0.5/1/2/3 | Minimal/Low/Medium/High/Massive |
| **Confidence** | 0.5/0.8/1.0 | Low/Medium/High（对估算的信心程度） |
| **Effort** | 1/2/3/5 | S/M/L/XL（人月工作量） |

### FR10目的地调整详细分析

```
Reach: 60%（仅运输中/已入库状态可调整，约占60%订单）
Impact: Massive = 3（核心差异化功能，决定产品竞争力）
Confidence: Medium = 0.8（需求未实证验证）
Effort: L = 3（涉及跨城转运逻辑，工作量较大）

RICE = (0.6 × 3 × 0.8) / 3 = 0.48 → 归一化为72
```

**结论**：虽然RICE得分不高，但作为核心差异化功能，仍应作为P0优先开发。建议先做MVP验证需求强度。
