> 本文档定义项目的分析目标、研究问题和预期产出，为后续 flow_design 和 execution_prompts 提供依据。

## 1. 数据概况摘要

| 属性 | 值 |
|------|-----|
| 数据来源 | 51job（前程无忧）招聘平台 |
| 文件名 | `前程无忧数据集.csv` |
| 数据规模 | 295 条有效招聘记录（原始 297 行，含 1 行重复表头） |
| 字段数量 | 10 列 |
| 数据格式 | CSV（UTF-8 编码） |
| 分析单元 | 每行 = 一条招聘信息（一个岗位 + 一个公司） |

### 1.1 字段清单

| 序号 | 原始字段名 | 语义说明 | 数据类型 | 示例值 |
|------|-----------|---------|---------|--------|
| 1 | 岗位 | 职位名称 | string | "Python开发工程师-CDN" |
| 2 | 公司 | 企业全称 | string | "华为技术有限公司" |
| 3 | 学历 | 学历要求 | string | "本科" / "大专" |
| 4 | 公司性质 | 企业所有制类型 | string | "民营公司" / "外资（非欧美）" |
| 5 | 薪资 | 薪资范围（含单位） | string | "1.8-3.5万/月" / "18-36万/年" |
| 6 | 学历要求 | **实为工作经验要求** | string | "1年经验" / "3-4年经验" |
| 7 | 工作经验 | **实为附加要求** | string | "英语良好" / 空值 |
| 8 | 公司规模 | 企业人员规模 | string | "10000人以上" / "50-150人" |
| 9 | 公司类型 | 行业标签（多值，空格分隔） | string | "计算机软件 通信/电信/网络设备" |
| 10 | 公司福利 | 福利标签（多值，短横线分隔） | string | "五险一金-通讯补贴-专业培训" |

### 1.2 已知数据质量问题

1. **第 2 行重复表头**：字段名与第 1 行略有不同，需剔除
2. **字段语义错位**：`学历要求` 列实际存储工作经验，`工作经验` 列实际存储附加要求
3. **薪资格式不统一**：存在"万/月"和"万/年"两种单位
4. **缺失值**：`工作经验`（附加要求）、`公司规模`、`公司类型`、`公司福利` 均有空值
5. **多值字段**：`公司类型`（空格分隔）、`公司福利`（短横线分隔）需拆分
6. **无显式城市列**：城市信息隐含在公司名称中，需提取

### 1.3 数据增强需求

| 需求 | 来源字段 | 方法 | 优先级 |
|------|---------|------|--------|
| 城市信息 | 公司 | 从公司名提取省市关键词 | P0 |
| 薪资数值 | 薪资 | 正则解析 + 单位统一换算 | P0 |
| 工作经验年数 | 学历要求 | 正则提取年数区间 | P0 |
| 主行业 | 公司类型 | 取第一个标签 | P1 |
| 福利标签列表 | 公司福利 | 短横线拆分 | P1 |

## 2. 业务背景

### 2.1 数据来源

51job（前程无忧）是中国领先的在线招聘平台，覆盖全国主要城市和行业。本数据集聚焦 Python 开发相关岗位的招聘信息，反映了该领域的人才市场需求特征。

### 2.2 分析单元

每条记录代表一条招聘信息，包含一个岗位名称、一个招聘公司、以及对应的学历要求、薪资范围、公司属性等信息。

### 2.3 目标受众

| 受众 | 关注点 | 期望产出 |
|------|--------|---------|
| **求职者** | 薪资水平、热门岗位、技能要求 | 择业参考：哪些城市/行业薪资更高、不同学历经验的薪资差异 |
| **企业HR** | 市场薪资基准、竞争对手招聘策略 | 招聘策略：如何制定有竞争力的薪资和福利方案 |
| **研究机构** | 人才市场趋势、供需结构 | 市场洞察：行业招聘热度变化、企业招聘偏好演变 |

### 2.4 核心业务问题

1. Python 开发岗位的薪资水平如何？受哪些因素影响最大？
2. 哪些城市/行业对 Python 开发人才需求最大？
3. 不同类型企业的招聘偏好（学历/经验门槛、福利）有何差异？
4. 福利完善程度与薪资水平是否存在相关性？

## 3. 分析目标清单

### 目标-1: 数据质量评估与清洗

- **类型**: 描述性分析
- **SMART表述**: 对 295 条原始数据进行全面质量评估，产出包含 295 条有效记录的清洗后数据集，薪资解析覆盖率 ≥ 95%，字段语义错位全部修正
- **优先级**: P0（前置依赖，所有后续分析的基础）
- **关键指标**: 有效记录数、缺失值比例、薪资解析成功率、字段重命名完成度
- **对应章节**: ch01

### 目标-2: 薪资水平多维分析

- **类型**: 描述性 + 诊断性分析
- **SMART表述**: 从学历、工作经验、行业、公司性质、公司规模、城市 6 个维度分析薪资影响因素，每个维度产出至少 1 张对比图表，识别薪资差异的统计显著性
- **优先级**: P0
- **关键指标**: 各维度薪资中位数、均值、分位数；维度间薪资差异幅度
- **对应章节**: ch02

### 目标-3: 人才市场供需结构刻画

- **类型**: 描述性分析
- **SMART表述**: 识别热门岗位 TOP20、城市招聘热度排名、行业需求分布，产出至少 3 张供需分布图表
- **优先级**: P0
- **关键指标**: 岗位招聘数量、城市招聘岗位占比、行业招聘占比
- **对应章节**: ch03

### 目标-4: 企业招聘偏好与福利特征分析

- **类型**: 描述性 + 诊断性分析
- **SMART表述**: 分析不同性质/规模企业的学历和经验门槛分布，提取 TOP15 福利标签并分析其行业分布，评估福利完善度与薪资水平的相关性
- **优先级**: P1
- **关键指标**: 学历/经验门槛分布比例、福利标签频次、福利-薪资相关系数
- **对应章节**: ch04

### 目标-5: 综合结论与决策建议

- **类型**: 规范性分析
- **SMART表述**: 汇总前 4 个章节的核心发现，面向求职者、企业、研究机构三类受众输出至少 10 条有数据支撑的结论和 5 条可执行建议
- **优先级**: P1
- **关键指标**: 结论数量、建议可执行性、数据引用完整性
- **对应章节**: ch05

## 4. 研究问题清单

### 核心研究问题（RQ-Core）

**问题**: 基于 51job 招聘数据，Python 开发岗位的薪资水平受哪些因素影响？人才市场供需结构和企业招聘偏好呈现什么特征？

**背景**: 求职者需要了解市场薪资水平以做出择业决策，企业需要了解市场行情以制定有竞争力的招聘策略，研究机构需要掌握人才市场趋势。

**数据支持**: 295 条 Python 开发岗位招聘记录，涵盖薪资、学历、经验、公司属性、行业、福利等多维信息。

**预期产出**: 薪资影响因素排序、热门岗位/城市/行业排名、企业招聘偏好画像、面向三类受众的决策建议。

### 子研究问题

#### RQ-Sub-1: 数据质量如何？有哪些需要清洗的问题？

- **与核心问题关系**: 基础性工作，为所有后续分析提供可靠数据
- **分析方法**: 数据质量评估（缺失值统计、异常值检测、格式一致性检查）
- **所需数据**: 原始 CSV 全部字段
- **产出形式**: 清洗后数据集 + 数据质量报告

#### RQ-Sub-2: 薪资的整体分布特征是什么？

- **与核心问题关系**: 描述性分析，建立薪资水平的基线认知
- **分析方法**: 描述统计（均值、中位数、分位数）、分布直方图、箱线图
- **所需数据**: 解析后的 salary_min / salary_max / salary_avg
- **产出形式**: 薪资分布图 + 分位数统计表

#### RQ-Sub-3: 学历、经验、行业、企业属性如何影响薪资？

- **与核心问题关系**: 诊断性分析，识别薪资的核心影响因素
- **分析方法**: 分组对比（箱线图、条形图）、多因素交叉分析
- **所需数据**: 学历、工作经验年数、行业、公司性质、公司规模、薪资
- **产出形式**: 6 维度薪资对比图 + 影响因素排序

#### RQ-Sub-4: 哪些岗位、城市、行业的需求最大？

- **与核心问题关系**: 描述性分析，刻画供需结构
- **分析方法**: 频次统计、TOP-N 排名、占比分析
- **所需数据**: 岗位名称、城市（提取）、行业标签
- **产出形式**: 热门岗位 TOP20 图、城市招聘热度图、行业需求分布图

#### RQ-Sub-5: 不同类型企业的招聘偏好和福利特征有何差异？

- **与核心问题关系**: 诊断性分析，揭示企业招聘策略差异
- **分析方法**: 分组统计、福利标签频次分析、关联分析
- **所需数据**: 公司性质、公司规模、学历要求、经验要求、福利标签、薪资
- **产出形式**: 企业招聘偏好画像 + 福利-薪资关联图

#### RQ-Sub-6: 福利完善程度与薪资水平是否相关？

- **与核心问题关系**: 诊断性分析，验证"高薪资企业福利也更好"的假设
- **分析方法**: 相关性分析、分组对比
- **所需数据**: 福利标签数量、薪资水平
- **产出形式**: 相关性分析结果 + 对比图

## 5. 分析范围与约束

### 5.1 数据范围

- **岗位范围**: Python 开发相关岗位（数据集已限定）
- **时间范围**: 数据采集时间未知（报告中需标注此局限性）
- **地理范围**: 全国（城市信息需从公司名提取，可能存在遗漏）

### 5.2 分析约束

| 约束 | 说明 | 应对策略 |
|------|------|---------|
| 样本量较小（295条） | 统计显著性有限 | 侧重描述性统计，避免过度推断 |
| 无时间维度 | 无法分析趋势变化 | 报告中标注，建议后续补充时序数据 |
| 城市信息需提取 | 可能存在提取不准确 | 建立省市关键词映射表，统计提取成功率 |
| 薪资格式不统一 | 需统一换算 | "万/年"÷12 统一为"万/月" |
| 仅 Python 岗位 | 结论不可泛化到全行业 | 报告中明确标注适用范围 |

### 5.3 分析排除条件

- 排除第 2 行重复表头数据
- 排除薪资字段为"面议"等无法解析的记录（统计占比）
- 排除城市提取结果为"未知"的记录（仅在城市维度分析时排除）
