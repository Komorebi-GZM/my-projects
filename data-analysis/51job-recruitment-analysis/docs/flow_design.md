## 第一章：研究概述

### 1.1 研究背景

51job（前程无忧）是中国领先的在线招聘平台，覆盖全国主要城市和行业，其招聘数据是观察人才市场供需结构的重要窗口。Python 作为近年来增长最快的编程语言之一，其开发岗位的薪资水平、技能要求、地域分布等特征备受求职者和企业关注。

本项目基于 51job 平台采集的 Python 开发相关岗位招聘数据（295 条有效记录），从**薪资维度、供需维度、企业特征维度**三个方向展开全维度分析，旨在为三类受众提供有数据支撑的决策参考：

- **求职者**：了解市场薪资水平、热门岗位、城市选择策略
- **企业 HR**：掌握市场薪资基准、竞争对手招聘策略、福利方案参考
- **研究机构**：获取人才市场趋势洞察、供需结构特征

## 第二章：基础数据概览（ch01）

> **章节原型**: A — 数据预处理型
> **章节标识**: ch01_data_overview
> **脚本文件**: `src/ch01_data_overview/overview.py` / `overview.ipynb`
> **输出目录**: `outputs/ch01_data_overview/`

### 2.1 研究目标

本章节是整个项目的基础性工作，目标是完成从原始数据到可分析数据的全流程转换。具体目标包括：

1. **数据质量评估**：全面诊断原始数据的缺失值、异常值、格式一致性问题
2. **字段语义修正**：修正"学历要求"与"工作经验"的字段语义错位
3. **薪资解析与标准化**：将文本格式的薪资范围统一解析为数值型（千/月）
4. **城市信息提取**：从公司名称中提取城市/地区信息
5. **多值字段拆分**：将"公司类型"和"公司福利"拆分为独立标签
6. **缺失值标记**：对缺失值进行统一标记（填充"未知"/"未标注"），保留记录完整性

### 2.2 数据输入

| 属性 | 说明 |
|------|------|
| 输入文件 | `data/前程无忧数据集.csv` |
| 加载方式 | `utils.data_loader.load_raw_data(skip_duplicate_header=True)` |
| 数据规模 | 295 条记录，10 列 |
| 编码格式 | UTF-8 |

**原始字段清单**：

| 序号 | 字段名 | 语义说明 | 数据类型 | 示例值 |
|------|--------|---------|---------|--------|
| 1 | 岗位 | 职位名称 | string | "Python开发工程师-CDN" |
| 2 | 公司 | 企业全称 | string | "华为技术有限公司" |
| 3 | 学历 | 学历要求 | string | "本科" / "大专" |
| 4 | 公司性质 | 企业所有制类型 | string | "民营公司" / "外资（非欧美）" |
| 5 | 薪资 | 薪资范围（含单位） | string | "1.8-3.5万/月" / "18-36万/年" |
| 6 | 学历要求 | **实为工作经验要求** | string | "1年经验" / "3-4年经验" |

| 10 | 公司福利 | 福利标签（多值，短横线分隔） | string | "五险一金-通讯补贴-专业培训" |

### 2.3 技术方法

| 方法 | 公式/参数 | 选择理由 |
|------|----------|---------|
| 重复表头剔除 | `df.iloc[1:]` 跳过第 2 行；判断依据：第二行首字段值为"职位"/"岗位"等表头关键词 | 原始数据第 2 行为重复表头，字段名与第 1 行略有不同，需剔除以保证数据一致性 |
| 字段重命名 | `df.rename(columns={'学历要求': '工作经验要求', '工作经验': '附加要求'})` | 原始字段名与实际语义不一致，重命名后消除歧义 |
| 薪资解析 | 正则表达式 `r'([\d.]+)-([\d.]+)(万/月\|万/年)'`；万/年换算：`value / DOMAIN_PARAMS['annual_to_monthly_divisor']`（即 ÷12）；输出列：`salary_min`, `salary_max`, `salary_avg`（单位：千/月） | 薪资格式不统一（万/月 vs 万/年），需正则提取数值并统一换算为千/月，便于后续数值分析 |
| 城市提取 | 遍历 `config.CITY_KEYWORDS`（50 个省市关键词），匹配公司名称中的关键词；未匹配到则标记为"未知" | 原始数据无显式城市列，城市信息隐含在公司名称中，需通过关键词匹配提取 |
| 多值字段拆分 | 公司类型：`str.split(' ')` 按空格拆分；公司福利：`str.split('-')` 按短横线拆分；取第一个标签作为主行业/主福利 | 多值字段不利于分组统计，拆分后可进行标签级别的频次分析和交叉分析 |
| 缺失值标记 | 字符串字段填充 `"未知"`；列表字段填充 `[]`（空列表）；不删除任何记录 | 295 条样本量较小，删除记录会损失信息；标记策略保留完整记录用于后续分析 |
| 数据质量报告 | `df.isnull().sum()` 统计缺失值；`df.dtypes` 检查类型；`df.duplicated().sum()` 检查重复 | 全面评估数据质量，为后续分析提供数据可靠性参考 |

### 2.4 实施步骤

**Step 2.1: 加载原始数据**

- 使用 `load_raw_data(skip_duplicate_header=True)` 加载原始 CSV
- 验证加载结果：记录数应为 295，列数应为 10
- 打印前 5 行和 `df.info()` 确认数据结构

**Step 2.2: 字段语义修正**

- 重命名字段：`学历要求` → `工作经验要求`，`工作经验` → `附加要求`
- 验证：打印重命名后的列名列表，确认无歧义

**Step 2.3: 薪资解析与标准化**

- 使用正则 `r'([\d.]+)-([\d.]+)(万/月|万/年)'` 提取薪资范围
- 对"万/年"格式的薪资除以 12，统一为"千/月"单位
- 生成 `salary_min`、`salary_max`、`salary_avg` 三列
- 统计薪资解析成功率（目标 >= 95%），记录无法解析的记录数
- 对无法解析的记录（如"面议"）的薪资列填充 NaN

**Step 2.4: 城市信息提取**

- 遍历 `config.CITY_KEYWORDS`（50 个省市关键词），对每条记录的公司名称进行匹配
- 生成 `city` 列，未匹配到则标记为"未知"
- 统计城市提取成功率（已知城市记录数 / 总记录数）

**Step 2.5: 多值字段拆分与主标签提取**

- 对 `公司类型` 列按空格拆分，生成 `industry_list` 列（列表类型），取第一个标签作为 `primary_industry`
- 对 `公司福利` 列按短横线拆分，生成 `benefit_list` 列（列表类型），取第一个标签作为 `primary_benefit`
- 统计拆分后的标签数量分布

**Step 2.6: 缺失值标记**

- 对所有字符串类型列的缺失值填充 `"未知"`
- 对列表类型列的缺失值填充 `[]`
- 生成缺失值统计表（各列缺失数量和占比）

**Step 2.7: 数据质量评估与产物输出**

- 生成数据质量报告（缺失值统计、字段类型、数据规模等）
- 绘制缺失值分布图
- 绘制各字段值分布图
- 保存清洗后数据集

## 第三章：薪资维度分析（ch02）

> **章节原型**: B — 分析探索型
> **章节标识**: ch02_salary_analysis
> **脚本文件**: `src/ch02_salary_analysis/salary.py` / `salary.ipynb`
> **输出目录**: `outputs/ch02_salary_analysis/`
> **前置依赖**: ch01（`outputs/ch01_data_overview/ch01_cleaned_data.csv`）

### 3.1 研究目标

本章节从 6 个核心维度深入分析薪资影响因素，建立对 Python 开发岗位薪资水平的全面认知。具体目标包括：

1. **薪资整体分布特征**：描述薪资的集中趋势、离散程度和分布形态
2. **学历-薪资关系**：分析不同学历要求对应的薪资差异
3. **工作经验-薪资关系**：分析不同经验年限对应的薪资差异
4. **行业-薪资关系**：分析不同行业（主行业）对应的薪资差异
5. **公司性质-薪资关系**：分析不同所有制类型企业的薪资差异
6. **公司规模-薪资关系**：分析不同规模企业的薪资差异
7. **城市-薪资关系**：分析不同城市对应的薪资差异

### 3.2 数据输入

| 属性 | 说明 |
|------|------|
| 输入文件 | `outputs/ch01_data_overview/ch01_cleaned_data.csv` |
| 加载方式 | `utils.data_loader.load_preprocessed(filepath)` |
| 关键字段 | `salary_min`, `salary_max`, `salary_avg`, `学历`, `工作经验要求`, `primary_industry`, `公司性质`, `公司规模`, `city` |
| 有效薪资记录 | 排除 salary_avg 为 NaN 的记录 |

### 3.3 技术方法

| 方法 | 公式/参数 | 选择理由 |
|------|----------|---------|
| 描述统计 | `df.groupby(dim)['salary_avg'].describe()`；输出 count, mean, std, min, 25%, 50%, 75%, max | 全面刻画各维度下的薪资分布特征，建立基线认知 |
| 分组对比（箱线图） | `seaborn.boxplot(data=df, x=dim, y='salary_avg', order=order)`；有序维度使用 `DOMAIN_PARAMS['education_order']` 和 `DOMAIN_PARAMS['company_size_order']` 排序 | 箱线图直观展示各组的薪资中位数、四分位距和离群值，便于比较组间差异 |
| 分组对比（条形图） | `utils.visualizer.plot_bar_chart(data, title, xlabel, ylabel)` | 条形图适合展示均值对比，补充箱线图无法展示的集中趋势 |
| 热力图 | `pd.pivot_table(values='salary_avg', index=dim1, columns=dim2, aggfunc='mean')` + `seaborn.heatmap` | 热力图适合展示两个维度的交叉薪资水平，识别高薪组合 |
| 排序分析 | 有序分类维度使用 `DOMAIN_PARAMS` 中预定义的顺序：`education_order = ['大专', '本科', '硕士', '博士']`，`company_size_order = ['少于50人', ..., '10000人以上']` | 学历和公司规模存在天然的大小关系，按序排列更符合业务直觉 |
| TOP-N 分析 | `df.groupby(dim)['salary_avg'].mean().sort_values(ascending=False).head(N)` | 识别薪资最高/最低的城市、行业等，提供简洁的排名信息 |

### 3.4 实施步骤

**Step 3.1: 加载清洗后数据**

- 使用 `load_preprocessed()` 加载 `ch01_cleaned_data.csv`
- 筛选有效薪资记录（`salary_avg` 非 NaN）
- 打印有效记录数和薪资基本统计（均值、中位数、标准差）

**Step 3.2: 薪资整体分布分析**

- 绘制薪资分布直方图（salary_avg），标注均值和中位数线
- 计算并展示薪资分位数表（P10, P25, P50, P75, P90）
- 识别薪资异常值（超出 1.5*IQR 范围）

**Step 3.3: 学历-薪资分析**

- 按学历分组计算薪资描述统计
- 绘制学历-薪资箱线图（按 `education_order` 排序）
- 绘制学历-薪资均值条形图
- 分析结论：学历对薪资的影响程度

**Step 3.4: 工作经验-薪资分析**

- 从"工作经验要求"字段提取经验年数（正则提取数字）
- 按经验年数分组计算薪资描述统计
- 绘制经验-薪资箱线图和条形图
- 分析结论：经验对薪资的影响程度

**Step 3.5: 行业-薪资分析**

- 按 `primary_industry` 分组计算薪资描述统计
- 绘制行业-薪资均值条形图（水平条形图，按薪资降序排列）
- 展示 TOP10 高薪行业

**Step 3.6: 公司属性-薪资分析**

- 按公司性质分组：绘制公司性质-薪资箱线图
- 按公司规模分组：绘制公司规模-薪资箱线图（按 `company_size_order` 排序）
- 绘制公司性质 × 公司规模交叉薪资热力图

**Step 3.7: 城市-薪资分析**

- 按城市分组计算薪资描述统计
- 绘制城市-薪资均值条形图（仅展示招聘岗位 >= 3 的城市）

| 行业-福利交叉 | `pd.crosstab(df['primary_industry'], top_benefits)` | 交叉表展示不同行业的福利偏好差异 |

- **求职者建议**（3-5 条）：城市选择、技能提升、薪资谈判策略
- **企业 HR 建议**（3-5 条）：薪资定价、福利方案、招聘渠道
- **研究机构建议**（2-3 条）：数据采集建议、研究方向建议

**Step 6.6: 研究局限性说明**

- 样本量限制（295 条，统计显著性有限）
- 时间维度缺失（无法分析趋势变化）
- 地域覆盖偏差（城市信息从公司名提取，可能遗漏）
- 岗位范围限定（仅 Python 开发岗位，不可泛化）
- 后续研究方向建议

**Step 6.7: 核心结论仪表板绘制**

- 设计仪表板布局（2x3 或 3x2 子图网格）
- 子图内容：薪资分布、学历-薪资对比、热门岗位 TOP10、城市热度、企业性质分布、福利-薪资关联
- 使用 matplotlib subplot 整合为一张综合图

**Step 6.8: 最终报告组装与产物输出**

- 按结构组装最终报告 Markdown 文档
- 保存最终报告和仪表板图片
