## 1. 版本概述

v0.1.1 是代码质量优化版本，主要聚焦于代码规范化、工具链统一和已知问题修复。本版本将代码格式化工具从 Black 迁移至 Ruff，统一了导入风格，并修复了测试中的期望值错误。

### 2.2 pyproject.toml 更新
```toml
[tool.ruff]
line-length = 120
target-version = "py312"

[tool.ruff.lint]
select = ["E", "F", "W", "I", "C4", "N", "UP", "RUF", "PT"]
ignore = [
    "E501",   # line too long (由 formatter 处理)
    "N801",   # lowercase function name (允许装饰器风格)
    "RUF001", # ambiguous unicode in strings (中文全角标点)
    "RUF002", # ambiguous unicode in docstrings
    "RUF003", # ambiguous unicode in comments
    "PT011",  # pytest.raises too broad
]
```

## 4. 代码规范 (Locked)

v0.1.1 之后，以下规范成为项目标准：

### 4.1 格式化与检查
- **唯一工具**: Ruff (formatter + linter)
- **行长度**: 120 字符
- **目标版本**: Python 3.12

### 4.2 导入风格
- **强制顶层导入**: 禁止函数内懒加载
- **分组顺序**: 标准库 → 第三方 → 本地

### 4.3 类型注解
- **类可变属性**: 必须使用 `typing.ClassVar`
- **函数参数/返回值**: 完整类型注解

### 4.4 中文支持
- **全角标点**: 允许使用（RUF001/002/003 已忽略）
- **文档语言**: 中文优先

### 4.5 架构约束
- **分层隔离**: GUI → Agent → LLM → Infra
- **禁止跨层**: 上层可调用下层，反之不可

## 5. FEN 行序规范 (重要)

v0.1.1 明确了 FEN 行序标准：

```python
# row=0: 黑方底线 (black back rank)
# row=9: 红方底线 (red back rank)

# FEN 字符串行序 (从上到下):
# row9 row8 row7 row6 row5 row4 row3 row2 row1 row0
```

**影响**: 所有 FEN 生成和解析必须遵循此规范。
