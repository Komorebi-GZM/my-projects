# 用户使用手册

## 文档元信息
- **文档版本**：v1.0
- **适用对象**：普通玩家、象棋爱好者、新手用户
- **应用类型**：桌面端中国象棋AI对弈工具
- **用途**：安装、启动、对局操作、个性化设置、常见问题指引

---

## 2. 软件简介与核心特色

### 2.1 软件定位

基于大语言模型的中国象棋AI对弈桌面工具，面向普通象棋爱好者和休闲玩家，提供接近人类思维的AI对弈体验。

### 2.2 核心特色

| 特色 | 说明 |
|-----|------|
| **智能对弈** | 基于DeepSeek/GPT/通义千问等大模型，走子风格接近人类 |
| **多模型切换** | 云端商用模型 + 本地开源模型自由切换，在线离线都能玩 |
| **标准规则** | 完整实现中国象棋规则，含将帅照面、马蹩脚、象塞眼、困毙等 |
| **悔棋重置** | 随时悔棋、重新开局，轻松练习 |
| **棋谱管理** | 自动保存对局、导出PGN棋谱、历史对局复盘 |
| **极简操作** | 鼠标点选走子，零学习成本 |

## 3. 系统环境与安装启动

### 3.1 系统要求

| 项目 | 要求 |
|-----|------|
| **操作系统** | Windows 10+ / macOS 12+ / Linux (Ubuntu 22.04+) |
| **内存** | 4 GB 及以上 |
| **磁盘空间** | 500 MB |
| **显示器** | 1280×720 及以上分辨率 |
| **网络** | 使用云端AI模型时需联网；本地模型无需联网 |

### 3.2 安装方式

#### 方式一：打包程序（推荐普通用户）

- 下载对应系统的安装包
- 双击安装，按向导完成
- 桌面生成快捷方式，双击即可启动

#### 方式二：源码运行（适合开发者）

- 获取项目代码
- 创建Python 3.12虚拟环境
- 安装依赖包
- 命令行启动应用

### 3.3 启动应用

- **Windows**：双击桌面快捷方式，或开始菜单中找到「中国象棋AI」
- **macOS**：在应用程序文件夹中双击启动
- **命令行**：激活虚拟环境后执行启动命令

## 4. 首次运行初始化配置

### 4.1 启动引导流程

首次启动应用时，自动进入配置向导，共3步：

```
步骤 1/3 ──→ 步骤 2/3 ──→ 步骤 3/3 ──→ 开始对弈
选择AI模型    配置API密钥    偏好设置
```

### 4.2 步骤1：选择AI模型

| 选项 | 说明 | 适用场景 |
|-----|------|---------|
| DeepSeek | 性价比最高，推荐首选 | 有网络连接 |
| OpenAI GPT-4o | 走子质量高 | 有网络连接 |
| 通义千问 | 国内访问稳定 | 有网络连接 |
| 本地Ollama | 完全离线，无需API | 无网络/隐私优先 |

### 4.3 步骤2：配置API密钥

- 选择云端模型后，需输入对应API密钥
- 密钥获取方式：
  - DeepSeek：访问 platform.deepseek.com 注册获取
  - OpenAI：访问 platform.openai.com 注册获取
  - 通义千问：访问 dashscope.console.aliyun.com 注册获取
- 选择本地模型（Ollama）则无需配置密钥

**连通性测试**：输入密钥后点击「测试连接」，显示 ✓ 表示配置成功。

### 4.4 步骤3：偏好设置

| 设置项 | 选项 | 说明 |
|-------|------|------|
| 先手选择 | 执红先行 / 执黑后行 | 红方先走是传统规则 |
| 棋盘大小 | 小(480) / 中(600) / 大(720) | 根据屏幕大小选择 |
| 走子动画 | 开启 / 关闭 | 关闭可提升低性能设备体验 |

完成设置后点击「开始对弈」进入主界面。

> 首次配置完成后，后续启动将直接进入主界面。如需修改配置，可在设置中随时调整。

## 6. 完整对局操作指南

### 6.1 基础走子操作

走子分三步完成：

```
第1步：选中棋子
  → 点击要移动的己方棋子
  → 棋子显示高亮边框
  → 可走位置显示提示标记

第2步：选择落点
  → 点击提示标记的合法位置
  → 棋子移动到目标位置

第3步：AI自动应答
  → 状态栏显示「AI思考中...」
  → AI完成走子后自动更新棋盘
  → 轮到玩家继续走子
```

### 6.2 取消选中

- 已选中棋子但不想走 → 点击棋盘空白区域取消选中
- 或点击另一个己方棋子切换选中

### 6.3 非法走子提示

| 情况 | 提示方式 |
|-----|---------|
| 点击对方棋子 | 无反应（无法选中） |
| 点击非合法落点 | 无反应，棋子不移动 |
| 走子后己方被将军 | 走子无效，提示「此走子会导致被将军」 |
| 走子违反特殊规则 | 走子无效，提示具体原因（如「马被蹩脚」） |

### 6.4 AI响应流程

| 阶段 | 状态栏显示 | 说明 |
|-----|----------|------|
| 等待AI | 「AI思考中...」 | AI正在计算走子 |
| AI走子完成 | 棋盘自动更新 | 显示AI的走子 |
| AI异常 | 「AI响应超时，使用备用走子」 | 降级到规则引擎随机走子 |
| 轮到玩家 | 「轮到你走子」 | 等待玩家操作 |

## 7. 游戏结束规则说明

### 7.1 胜负判定

| 结束方式 | 说明 | 判定结果 |
|---------|------|---------|
| **将死** | 一方被将军且无法解除 | 被将死方输 |
| **困毙** | 轮到一方走子但无合法走子 | 无合法走子方输 |

### 7.2 和棋判定

| 和棋方式 | 说明 |
|---------|------|
| **长将循环** | 一方连续将军超过规定次数，判和 |
| **自然限着** | 连续60步未吃子，自动判和 |
| **协议和棋** | 任一方主动提出和棋，对方同意 |

### 7.3 对局结束弹窗

对局结束时自动弹出结果弹窗：

```
┌─────────────────────────────┐
│        对局结束              │
│                             │
│     🏆 你赢了！              │
│     （将死 / 对方认输）      │
│                             │
│  总步数：42步                │
│  用时：8分32秒               │
│                             │
│  [再来一局]  [查看棋谱]  [退出] │
└─────────────────────────────┘
```

弹窗按钮说明：

| 按钮 | 功能 |
|-----|------|
| 再来一局 | 以相同配置开始新对局 |
| 查看棋谱 | 进入棋谱复盘界面 |
| 退出 | 关闭弹窗，返回棋盘（可继续查看） |

## 8. 悔棋/重置等常用功能

### 8.1 悔棋

| 项目 | 说明 |
|-----|------|
| **功能** | 撤销最近一步走子（包含玩家和AI各一步） |
| **操作** | 点击底部「悔棋」按钮 |
| **次数限制** | 不限次数，可连续悔棋至开局 |
| **状态影响** | 棋盘回退两步（玩家一步+AI一步），走子记录同步更新 |

> 悔棋会同时撤销AI的应答走子，确保轮次正确。

### 8.2 重置对局

| 项目 | 说明 |
|-----|------|
| **功能** | 清空当前棋盘，重新开始 |
| **操作** | 点击底部「重置」按钮 |
| **确认** | 弹出确认对话框，防止误操作 |
| **先手选择** | 重置后可选择切换先手方 |

### 8.3 退出与自动保存

| 项目 | 说明 |
|-----|------|
| **自动保存** | 对局过程中每步自动保存，关闭窗口不丢失进度 |
| **未完成对局** | 下次启动时检测到未完成对局，提示「恢复上局」或「开始新局」 |
| **已完成对局** | 自动保存到历史记录，可在「历史」中查看 |

## 9. 个性化设置功能

### 9.1 设置入口

- 主界面底部点击「设置」按钮
- 或使用快捷键 `Ctrl + ,`

### 9.2 AI模型设置

| 设置项 | 说明 | 可选值 |
|-------|------|-------|
| AI提供商 | 模型服务厂商 | DeepSeek / OpenAI / 通义千问 / Ollama |
| AI模型 | 具体模型名称 | 根据提供商动态显示 |
| API密钥 | 访问凭证 | 输入框 + 测试连接按钮 |
| 创造性参数 | AI走子多样性 | 0.0（保守）~ 1.0（激进），默认0.1 |
| 响应超时 | 等待AI的最长时间 | 1~10秒，默认3秒 |

### 9.3 棋盘界面设置

| 设置项 | 说明 | 可选值 |
|-------|------|-------|
| 界面主题 | 棋盘视觉风格 | 经典 / 木质 / 简约 |
| 棋盘大小 | 棋盘像素尺寸 | 480 / 600 / 720 |
| 显示坐标 | 棋盘边缘坐标标注 | 开启 / 关闭 |
| 走子动画 | 棋子移动动画效果 | 开启 / 关闭 |
| 音效 | 走子/吃子音效 | 开启 / 关闭 |

### 9.4 对局规则设置

| 设置项 | 说明 | 可选值 |
|-------|------|-------|
| 默认先手 | 每局默认执红或执黑 | 执红先行 / 执黑后行 |
| 长将判和 | 连续将军次数上限 | 6次 / 10次 / 不限制 |
| 自然限着 | 未吃子和棋步数 | 40步 / 60步 / 80步 |
| 自动保存 | 每步自动保存对局 | 开启 / 关闭 |

### 9.5 高级设置

| 设置项 | 说明 |
|-------|------|
| 日志级别 | 调试信息详细程度（普通用户保持默认INFO） |
| 数据目录 | 数据库和备份文件存储位置 |
| 清空历史 | 删除所有历史对局记录（需二次确认） |
| 重置配置 | 恢复所有设置为默认值（需二次确认） |

## 10. 棋谱保存、加载与复盘

### 10.1 对局保存

| 保存方式 | 触发条件 | 说明 |
|---------|---------|------|
| **自动保存** | 每步走子后自动触发 | 无需手动操作，关闭不丢数据 |
| **手动保存** | 对局中随时可保存 | 通过菜单或快捷键触发 |

### 10.2 历史对局浏览

- 点击主界面「历史」按钮进入历史记录
- 列表显示所有对局记录：

| 显示项 | 说明 |
|-------|------|
| 对局日期 | 开始时间 |
| 对局结果 | 胜/负/和/未完成 |
| AI模型 | 使用的AI模型名称 |
| 总步数 | 对局总走子数 |

- 点击某条记录可进入复盘模式

### 10.3 棋谱复盘

进入复盘模式后：

| 操作 | 说明 |
|-----|------|
| 查看走子 | 逐步前进/后退浏览每步走子 |
| 跳转到开头 | 回到初始局面 |
| 跳转到结尾 | 跳到最终局面 |
| 自动播放 | 自动逐步播放走子过程 |
| 退出复盘 | 返回历史列表 |

### 10.4 棋谱导出

| 导出格式 | 说明 |
|---------|------|
| **PGN** | 标准棋谱格式，可用第三方象棋软件打开复盘 |
| **JSON** | 结构化数据格式，包含完整走子和FEN信息 |

导出操作：在历史记录中选中对局 → 点击「导出」→ 选择格式 → 保存到本地文件。

## 12. 常见问题FAQ

### 12.1 AI相关

**Q：AI走子很慢怎么办？**

A：可能原因及解决方案：
- 网络延迟：切换到更快的模型（如DeepSeek）
- 模型响应慢：在设置中降低响应超时时间，超时后自动使用备用走子
- 使用本地模型：确保GPU显存足够，或切换到更小的模型（如Qwen2.5 3B）

**Q：AI走子看起来不太聪明？**

A：大模型擅长理解局势但非专业象棋引擎，走子质量取决于模型能力。可尝试：
- 切换更强的模型（GPT-4o > DeepSeek > 通义千问）
- 适当提高创造性参数（0.1→0.3），增加走子多样性

**Q：AI一直使用备用走子（不走大模型）？**

A：说明大模型调用持续失败，请检查：
- API密钥是否正确且未过期
- 网络连接是否正常
- 查看日志文件获取详细错误信息

### 12.2 配置相关

**Q：API密钥在哪里获取？**

A：
- DeepSeek：platform.deepseek.com 注册后创建
- OpenAI：platform.openai.com 注册后创建
- 通义千问：dashscope.console.aliyun.com 注册后创建

**Q：配置密钥后测试连接失败？**

A：
- 确认密钥复制完整，无多余空格
- 确认网络可以访问对应API地址
- 如使用代理，确认代理配置正确

**Q：如何切换到本地离线模型？**

A：
- 先安装Ollama（ollama.com），下载并启动模型
- 在设置中将AI提供商切换为「Ollama」
- 无需配置API密钥，完全离线运行

### 12.3 界面相关

**Q：棋子显示为方块/乱码？**

A：
- 确认系统已安装中文字体
- Windows：确认系统字体未损坏
- 尝试在设置中切换界面主题

**Q：窗口太大/太小？**

A：
- 在设置中调整棋盘大小（480/600/720）
- 使用 `F11` 切换全屏模式
- 调整操作系统显示缩放比例

**Q：走子动画卡顿？**

A：
- 在设置中关闭「走子动画」
- 缩小棋盘尺寸
- 关闭其他占用GPU的应用程序

### 12.4 数据与安全

**Q：对局记录会丢失吗？**

A：不会。每步走子自动保存到本地数据库，关闭窗口不丢失。但删除应用数据目录会导致记录丢失。

**Q：API密钥安全吗？**

A：
- 密钥保存在本地配置文件中，不会上传到任何第三方
- 日志中密钥自动脱敏，仅显示前4位
- 建议定期更换密钥，不要将密钥分享给他人

**Q：如何删除所有对局记录？**

A：在设置 → 高级设置 → 点击「清空历史」（需二次确认）。
