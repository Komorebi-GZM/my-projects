# 部署与运维文档

## 文档元信息
- **文档版本**：v1.0
- **适配技术栈**：Python 3.12 / SQLite / Pygame 2.6 / LangChain 1.2+
- **模块名称**：部署运维模块
- **应用类型**：桌面应用（无需服务器部署）
- **适用角色**：系统管理员、开发运维人员、自托管最终用户

## 2. 文档概述与应用定位

### 2.1 文档用途

本文档指导不同角色完成以下运维任务：

| 角色 | 核心任务 |

| **开发运维** | 环境搭建、性能调优、日志分析、版本升级 |

### 2.2 应用定位

```
┌─────────────────────────────────────────────┐
│         中国象棋AI对弈工具                    │
│         桌面应用 · 无需服务器部署              │
├─────────────────────────────────────────────┤

│  AI能力：云端大模型API + 本地Ollama可选        │
```

## 4. 安装部署步骤

### 4.1 部署流程总览

```
步骤1: 获取代码
    │
    ▼
步骤2: 创建虚拟环境
    │
    ▼
步骤3: 安装依赖包
    │
    ▼
步骤4: 配置大模型API
    │
    ▼
步骤5: 安装验证
    │
    ▼
步骤6: 首次启动
```

### 4.2 步骤详解

#### 步骤1：获取代码

- 从版本库拉取项目代码到本地目录
- 确认目录结构完整（src/、config/、assets/等）

#### 步骤2：创建虚拟环境

- 在项目根目录创建Python 3.12虚拟环境
- 激活虚拟环境（Windows/macOS/Linux命令不同）
- 验证Python版本为3.12.x

#### 步骤3：安装依赖包

- 使用pip从requirements.txt安装全部依赖
- 可选：安装requirements-dev.txt（开发/测试额外依赖）
- 验证关键包安装成功（pygame、langchain、langgraph）

#### 步骤4：配置大模型API

根据使用场景选择配置方式：

| 方式 | 适用场景 | 说明 |
|-----|---------|------|
| **环境变量** | 开发/运维 | 在.env文件中配置API Key |
| **GUI设置** | 最终用户 | 首次启动向导中配置 |
| **配置文件** | 高级用户 | 在config/app.yaml中指定 |

需配置的核心项：
- API Key（DeepSeek/OpenAI/通义，至少配置一个）
- API Base URL（默认值通常可用，私有部署需修改）
- 默认模型名称

#### 步骤5：安装验证

- 运行验证脚本，检查各模块加载正常
- 验证项：Python版本、依赖包、数据库初始化、GUI渲染

#### 步骤6：首次启动

- 执行启动命令，确认GUI窗口正常显示
- 完成首次运行向导（模型选择、API连接测试、偏好设置）

## 7. 日志管理规范

### 7.1 日志存储位置

```
logs/
├── app.log                 # 应用主日志
├── app.log.1               # 轮转备份
├── llm_calls.log           # LLM调用专用日志
└── llm_calls.log.1         # 轮转备份
```

### 7.2 日志分类

| 日志类型 | 文件 | 记录内容 |
|---------|------|---------|
| **应用日志** | app.log | 启动/关闭、对局事件、异常错误、模块加载 |
| **LLM调用日志** | llm_calls.log | 每次LLM请求/响应、Token消耗、延迟、解析结果 |

### 7.3 日志格式

```
应用日志格式：
[2026-05-13 10:30:00.123] [INFO ] [game_controller] 对局创建成功 game_id=a1b2c3d4

LLM调用日志格式：
[2026-05-13 10:30:01.456] [DEBUG] [llm_module] provider=deepseek model=deepseek-chat
  prompt_tokens=180 completion_tokens=4 latency_ms=1250
  raw_output="h2e2" parsed_move="h2e2" is_valid=True
```

### 7.4 日志级别使用规范

| 级别 | 使用场景 | 示例 |
|-----|---------|------|
| **DEBUG** | 详细调试信息 | LLM请求/响应原文、内部状态变更 |
| **INFO** | 关键业务事件 | 对局开始/结束、走子记录、配置加载 |
| **WARNING** | 可恢复异常 | LLM超时降级、解析失败重试、API限流 |
| **ERROR** | 不可恢复错误 | 数据库损坏、API Key无效、致命异常 |

### 7.5 日志轮转策略

| 参数 | 值 | 说明 |
|-----|---|------|
| 单文件最大 | 10 MB | 超过后自动轮转 |
| 备份保留数 | 5 | 最多保留5个历史文件 |
| 轮转方式 | 按大小 | 不按时间轮转 |

## 8. 数据库运维

### 8.1 数据库文件

| 属性 | 说明 |
|-----|------|
| **默认路径** | `data/chess_ai.db` |
| **文件格式** | SQLite 3 单文件 |
| **包含内容** | 业务表（games/moves/settings）+ LangGraph检查点表 |

### 8.2 备份策略

#### 自动备份

| 配置项 | 默认值 | 说明 |
|-------|-------|------|
| 触发条件 | 每10局对弈 | 可配置 |
| 备份路径 | `data/backups/` | 自动创建 |
| 命名规则 | `chess_ai_YYYYMMDD_HHMMSS.db` | 时间戳命名 |
| 保留数量 | 最近10份 | 自动清理旧备份 |

#### 手动备份

- 复制数据库文件到目标位置即可（SQLite单文件设计）
- 备份前建议执行WAL checkpoint确保数据一致性
- 备份期间应用可继续运行（WAL模式支持并发读取）

### 8.3 恢复流程

```
步骤1: 关闭应用（确保无写入操作）
    │
    ▼
步骤2: 将备份文件复制到 data/chess_ai.db
    │
    ▼
步骤3: 启动应用，验证数据完整性
    │
    ▼
步骤4: 检查对局列表、走子记录是否完整
```

### 8.4 数据库维护

| 维护项 | 操作方式 | 频率 |
|-------|---------|------|
| **旧检查点清理** | 应用内自动清理（保留最近N局） | 每次启动时 |
| **数据库压缩** | 执行VACUUM命令回收空间 | 手动/每月 |
| **完整性检查** | 执行PRAGMA integrity_check | 手动/异常后 |
| **孤立数据清理** | 删除无对局关联的走子记录 | 手动 |

## 9. 性能调优策略

### 9.1 LLM延迟优化

| 优化方向 | 具体措施 | 预期效果 |
|---------|---------|---------|
| **模型选择** | 选用响应快的模型（DeepSeek > GPT-4o） | 延迟降低30-50% |
| **参数调整** | 降低temperature（0.1→0.0）、减少max_tokens | 延迟降低10-20% |
| **本地部署** | Ollama部署Qwen2.5 3B/7B | 零网络延迟，但需GPU |
| **超时设置** | 合理设置timeout（推荐3s），快速降级 | 避免长时间等待 |
| **连接复用** | 复用HTTP连接，减少握手开销 | 延迟降低5-10% |

### 9.2 Token消耗控制

| 优化方向 | 具体措施 | 预期效果 |
|---------|---------|---------|
| **Prompt版本** | 云端用极简版（≤220 Token） | Token降低40% |
| **上下文精简** | 仅传当前FEN，不传历史对局 | Token降低60%+ |
| **示例数量** | 严格限制≤2个Few-shot示例 | Token降低30% |
| **输出约束** | 强制仅输出坐标，禁止解释 | 输出Token≤10 |

### 9.3 GUI性能优化

| 优化方向 | 具体措施 | 适用场景 |
|---------|---------|---------|
| **关闭动画** | ui.animation_enabled=false | 低性能设备 |
| **降低分辨率** | ui.board_size=480 | 小屏幕/低性能 |
| **关闭音效** | ui.sound_enabled=false | 无需音效时 |
| **脏矩形重绘** | 仅重绘变化区域（默认启用） | 所有场景 |

### 9.4 数据库性能优化

| 优化方向 | 具体措施 | 说明 |
|---------|---------|------|
| **WAL模式** | 默认启用 | 读写并发，写入不阻塞读取 |
| **索引优化** | 已预建核心索引 | 覆盖主要查询路径 |
| **定期VACUUM** | 每月执行一次 | 回收已删除数据空间 |
| **批量写入** | 走子记录批量提交 | 减少事务开销 |

## 10. 常见故障排查

### 10.1 启动故障

| 故障现象 | 可能原因 | 排查步骤 |
|---------|---------|---------|
| `ModuleNotFoundError` | 依赖未安装 | 检查虚拟环境是否激活，重新安装依赖 |
| `Python版本不匹配` | Python非3.12 | 执行`python --version`确认版本 |
| `ImportError: pygame` | Pygame未安装 | 执行`pip install pygame` |
| `显示设备初始化失败` | 无图形界面/SSH环境 | 确认本地显示器可用，不可用SSH无图形模式 |

### 10.2 API配置故障

| 故障现象 | 可能原因 | 排查步骤 |
|---------|---------|---------|
| `AuthenticationError` | API Key无效/过期 | 检查.env中Key是否正确，重新生成 |
| `ConnectionError` | 网络不通/代理问题 | 检查网络连通性，确认API端点可达 |
| `RateLimitError` | API调用限流 | 降低请求频率，检查账户配额 |
| `模型不存在` | 模型名称拼写错误 | 对照config/models.yaml确认模型名 |

### 10.3 LLM响应故障

| 故障现象 | 可能原因 | 排查步骤 |
|---------|---------|---------|
| AI走子始终为规则引擎兜底 | LLM持续超时/解析失败 | 查看llm_calls.log，检查API连通性 |
| AI走子非法 | 解析器提取错误坐标 | 查看raw_output，检查解析器正则 |
| AI走子很慢 | 模型响应延迟高 | 切换更快的模型或降低timeout |
| AI输出格式混乱 | Prompt版本不匹配 | 确认prompt_version与模型适配 |

### 10.4 数据库故障

| 故障现象 | 可能原因 | 排查步骤 |
|---------|---------|---------|
| `database is locked` | 并发写入冲突 | 确认无多实例运行，检查WAL模式 |
| 对局数据丢失 | 数据库文件损坏 | 从备份恢复，执行integrity_check |
| 磁盘空间不足 | 数据库/日志文件过大 | 清理旧备份和日志，执行VACUUM |

### 10.5 GUI显示故障

| 故障现象 | 可能原因 | 排查步骤 |
|---------|---------|---------|
| 棋子图片缺失 | assets目录不完整 | 确认assets/images/pieces/下14张图片存在 |
| 窗口显示异常 | 分辨率不兼容 | 调整board_size配置，尝试480/600/720 |
| 中文乱码 | 字体缺失 | 确认系统中文字体可用 |

### 10.6 调试模式

启用调试模式获取详细日志：

```bash
python -m src.main --debug
```

调试模式效果：
- 日志级别降至DEBUG
- LLM请求/响应原文完整记录
- 异常堆栈完整输出
- GUI窗口显示FPS计数器

## 11. 安全运维建议

### 11.1 API密钥安全

| 措施 | 说明 |
|-----|------|
| **.env文件不入库** | .gitignore中已配置排除.env |
| **禁止硬编码** | API Key必须通过环境变量或配置文件传入 |
| **禁止日志打印** | 日志中API Key自动脱敏（仅显示前4位） |
| **定期轮换** | 建议每90天更换一次API Key |
| **最小权限** | API Key仅授予必要权限，禁止全量访问 |

### 11.2 本地数据库安全

| 措施 | 说明 |
|-----|------|
| **文件权限** | 数据库文件权限设为仅当前用户可读写 |
| **备份加密** | 敏感对局数据备份时可加密存储 |
| **传输安全** | 数据库文件不通过网络传输（本地应用） |

### 11.3 日志安全

| 措施 | 说明 |
|-----|------|
| **API Key脱敏** | 日志中自动替换为 `sk-****xxxx` |
| **LLM输出脱敏** | 原始输出仅记录于DEBUG级别日志 |
| **日志级别控制** | 生产环境使用INFO级别，避免敏感信息泄露 |
| **定期清理** | 日志轮转自动清理旧文件 |

### 11.4 网络安全

| 措施 | 说明 |
|-----|------|
| **HTTPS强制** | 所有API调用使用HTTPS协议 |
| **代理支持** | 支持通过HTTP_PROXY/HTTPS_PROXY代理访问 |
| **无外部端口** | 应用不开放任何网络端口（纯客户端） |

## 14. 可选：Docker容器化部署

### 14.1 适用场景

| 场景 | 说明 |
|-----|------|
| **API服务模式** | 仅暴露AI走子API，无GUI |

| **CI/CD测试** | 自动化测试环境标准化 |

> 注意：GUI模式不支持容器化部署（需要显示器），容器化仅适用于无头API服务模式。

### 14.2 核心部署逻辑

#### 镜像构建

- 基础镜像：Python 3.12-slim
- 安装系统依赖（Pygame所需的SDL库）
- 复制项目代码和配置文件
- 安装Python依赖
- 设置入口点

#### 环境变量配置

容器运行时通过环境变量注入配置：

| 变量 | 说明 |
|-----|------|
| `AI_PROVIDER` | 模型提供商 |
| `AI_MODEL` | 模型名称 |
| `DEEPSEEK_API_KEY` | API密钥 |
| `DB_PATH` | 数据库挂载路径 |
| `LOG_LEVEL` | 日志级别 |

#### 运行命令

- 挂载数据卷（数据库持久化）
- 映射API端口（如使用API模式）
- 设置环境变量

### 14.3 GUI模式注意事项

- 容器内运行GUI需要X11转发或VNC
- 推荐直接在宿主机运行GUI模式
- 容器化仅推荐用于API服务或无头测试
