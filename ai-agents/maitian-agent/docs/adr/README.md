# 架构决策记录 (Architecture Decision Records)

本目录存放麦田智囊项目的所有架构决策记录（ADR）。

## 什么是 ADR

ADR 是记录架构决策的轻量级文档，记录：
- 为什么做这个决策
- 上下文和考虑因素
- 做了哪个决策
- 决策的影响

## 使用方式

每个 ADR 是一个独立的 Markdown 文件，按编号命名：

```
docs/adr/
├── README.md          # 本文件
├── 0001-use-langchain.md  # ADR 示例
└── 0002-use-chromadb.md
```

## ADR 模板

```markdown
# [编号] 标题

- 状态: [提议/接受/废弃/取代]
- 日期: YYYY-MM-DD

## 背景

<!-- 为什么需要做这个决策 -->

## 决策

<!-- 我们做了什么选择 -->

## 理由

<!-- 为什么做这个选择，有哪些考虑因素 -->

## 影响

<!-- 选择后的正面和负面影响 -->

## 备选方案

<!-- 还考虑了哪些方案，为什么不选 -->
```
