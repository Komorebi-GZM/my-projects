## 二、第一阶段：问题诊断与修复（原型期）

### 2.1 问题发现

运行 Demo 时出现模块导入错误：

```
ModuleNotFoundError: No module named 'langchain.prompts'
```

### 2.2 原因分析

- LangChain 1.2.15 版本中，`prompts` 模块已迁移至 `langchain_core`
- PaddleOCR 无法在当前 Python 环境中安装

### 2.3 解决方案

1. **导入路径迁移**：`from langchain.prompts` → `from langchain_core.prompts`
2. **OCR 方案替换**：PaddleOCR → EasyOCR（兼容性更好）

### 2.4 修改文件

- `demo/agents/quick_lesson_prep.py`
- `demo/agents/wisdom_transfer.py`

## 三、第二阶段：v1.0.0 框架搭建（框架期）

### 3.1 架构设计

基于四层架构设计，创建完整的主项目框架：

```
感知层（OCR/ASR）→ RAG 层（知识库/检索）→ 记忆层（对话/画像）→ 执行层（6 Agent）
```

### 3.2 目录结构创建

```
maitian_agent/
├── agents/              # 6 大核心 Agent + BaseAgent + AgentFactory
├── rag/                 # RAG 知识库（嵌入/向量存储/知识库）
├── memory/              # 记忆系统（对话记忆/教师画像）
├── tools/               # 感知层（OCR/ASR/文件处理）
├── api/                 # FastAPI REST API
├── frontend/            # Streamlit Web UI
├── config/              # 配置管理（Pydantic Settings）
└── cli.py               # CLI 入口
```

### 3.3 核心模块

| 模块 | 功能 | 实现方式 |
|------|------|---------|
| BaseAgent | 抽象基类 | ABC + `@abstractmethod` build_chain/run |
| AgentFactory | 工厂模式 | `_AGENT_REGISTRY` 注册表 + `create_all()` |
| KnowledgeBase | 三层知识库 | 通用/学校/教师层级 + 向量检索 |
| ConversationMemory | 对话记忆 | InMemoryChatMessageHistory + JSON 持久化 |
| TeacherProfileManager | 教师画像 | JSON 文件存储 |

## 四、第三阶段：P0–P3 全面重构（重构期）

这是项目最关键的阶段，通过 TDD 红绿重构循环，将原型框架升级为生产级架构。

### 4.1 P0 — 生产阻塞（3 项）

**目标**：所有入口统一使用 AgentFactory，消除直接实例化 Agent 的代码。

| 任务 | 改动 | 测试 |
|------|------|------|
| Frontend 接入 AgentFactory | `streamlit_app.py` 使用 `factory.create_all()` | 12 项 |
| CLI 接入 AgentFactory | `cli.py` 使用 `_create_factory()` | 8 项 |
| API 接入 AgentFactory | `routes.py` 使用 lifespan + `app.state.agents` | 9 项 |

**关键设计决策**：
- FastAPI 使用 `@asynccontextmanager lifespan` 替代已废弃的 `on_event("startup")`
- Streamlit 通过 `st.session_state.agents` 缓存 Agent 实例
- CLI 通过 `_get_agents()` 延迟创建，支持命令级复用

### 4.2 P1 — 架构完善（4 项）

**目标**：依赖注入统一、RAG 深度集成、路由记忆保存。

| 任务 | 改动 | 测试 |
|------|------|------|
| 注入方式统一 | 7 种依赖从属性赋值改为构造函数 kwargs | 51 项 |
| MaterialAgent RAG 深度集成 | `{reference_section}` 模板变量 | 17 项 |
| RouterAgent 记忆保存 | `_save_to_memory()` + `_format_output()` | 13 项 |
| FastAPI lifespan 迁移 | `@asynccontextmanager` + `app.state` | 9 项 |

**关键设计决策**：
- BaseAgent 构造函数接收 `llm`、`settings`、`knowledge_base`、`conversation_memory`、`teacher_profile`、`ocr_reader`、`asr` 共 7 种依赖
- AgentFactory 通过 `_build_kwargs()` 统一组装依赖，传入各 Agent 的 `super().__init__(**kwargs)`
- MaterialAgent 的 `recommend_materials()` 返回 `Tuple[str, bool]`，区分有/无 RAG 结果

### 4.3 P2 — 功能增强（4 项）

**目标**：BM25 混合检索、本地 ASR、类型注解统一、记忆并发安全。

| 任务 | 改动 | 测试 |
|------|------|------|
| BM25 混合检索 | `BM25Retriever` + `_rrf_fusion()` + `HybridRetriever` | 17 项 |
| LocalWhisperASR 本地模型 | openai-whisper 懒加载 + `transcribe/transcribe_with_srt` | 18 项 |
| LLM 类型注解统一 | `Optional[ChatOpenAI]` → `Optional[BaseChatModel]` | 29 项 |
| ConversationMemory 文件锁 | `_FileLock(fcntl.flock)` + 超时降级 | 17 项 |

**关键设计决策**：
- BM25 使用 jieba 分词 + rank_bm25 实现，与向量检索通过 RRF（Reciprocal Rank Fusion）加权融合
- LocalWhisperASR 采用懒加载模式，`_model` 属性在首次调用 `transcribe()` 时才加载模型
- 6 个 Agent 的 `llm` 参数类型统一为 `Optional[BaseChatModel]`，`build_chain()` 开头添加 `assert self.llm is not None`
- `_FileLock` 基于 `fcntl.flock(LOCK_EX | LOCK_NB)` 实现跨进程互斥，超时后返回 `acquired=False` 自动降级

### 4.4 P3 — 工程化（3 项）

**目标**：CI 流水线、demo 目录同步、配置单例优化。

| 任务 | 改动 | 测试 |
|------|------|------|
| GitHub Actions CI | 3 Job（ruff + pytest-cov + mypy） | — |
| demo/ Legacy 标记 | 4 文件 `[LEGACY]` docstring + README 重写 | — |
| Settings 单例优化 | `__init__.py`: `Settings()` → `get_settings()` | 11 项 |

**关键设计决策**：
- CI 使用 `concurrency` + `cancel-in-progress` 避免重复运行
- ruff 规则集选用 E/W/F/I/B（逐步启用 UP/SIM）
- 覆盖率阈值设为 75%，排除 `__init__.py`、CLI、前端、OCR/文件工具
- `__init__.py` 的 `settings = get_settings()` 确保包级/settings.py/config 三处单例一致

## 五、TDD 开发方法

整个 P0–P3 重构阶段严格遵循 TDD 红绿重构循环：

### 5.1 开发流程

```
RED   → 编写测试，确认失败（验证测试有效）
GREEN → 编写最小实现，确认通过（验证功能正确）
REFACTOR → 重构代码，确认全量测试通过（验证无回归）
```

### 5.2 测试策略

- **接口测试**：通过公共 API（`run()`、`save_context()`）验证行为，不测试私有方法
- **Mock 策略**：使用 `unittest.mock.patch` 替换外部依赖（LLM、OCR、向量存储）
- **AST 验证**：`test_agent_type_annotation.py` 通过 AST 解析验证类型注解，而非字符串匹配
- **并发测试**：`test_memory_concurrent_safe.py` 使用 `threading.Barrier` + `ThreadPoolExecutor` 验证线程安全

### 5.3 测试增长曲线

| 阶段 | 累计测试数 | 新增测试文件 |
|------|-----------|-------------|
| 框架期结束 | ~50 | 基础集成测试 |

## 六、技术决策记录（ADR）

### ADR-001：四层 Agent 架构设计

- **决策**：采用感知层→RAG 层→记忆层→执行层的四层架构
- **理由**：关注点分离，各层可独立测试和替换
- **文档**：`docs/adr/0001-four-layer-agent-architecture.md`

### ADR-002：Agent 接口统一

- **决策**：所有 Agent 的 `run()` 统一为 `run(input_data: Dict) -> Dict`
- **理由**：RouterAgent 需要统一调度接口，Dict 格式支持结构化元数据
- **文档**：`docs/adr/0002-agent-interface-unification.md`

### ADR-003：依赖注入模式

- **决策**：7 种依赖通过构造函数 kwargs 注入，AgentFactory 统一管理
- **理由**：可测试性（mock 替换）、灵活性（不同配置创建不同 Agent）
- **文档**：`docs/adr/0003-dependency-injection-pattern.md`

### 其他关键决策

| 决策 | 选择 | 替代方案 | 理由 |
|------|------|---------|------|
| OCR 方案 | EasyOCR | PaddleOCR | 环境兼容性 |
| LLM 调用模式 | LCEL 管道 | 命令式 invoke | 声明式、可缓存、可组合 |
| 配置管理 | Pydantic Settings | os.getenv 散落 | 类型安全、验证、.env 自动加载 |
| 文件锁 | fcntl.flock | threading.Lock | 跨进程安全 |
| CI 工具 | ruff | black + flake8 + isort | 单工具替代三工具，更快 |
| 向量存储 | ChromaDB | Milvus | 轻量级，无需独立服务 |
