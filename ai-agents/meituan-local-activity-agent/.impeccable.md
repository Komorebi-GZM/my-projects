## Design Context

### Users
Chinese consumers planning local leisure activities (dining, entertainment, outings). They input a natural language description of what they want (time, group, budget, preferences) and get AI-generated, multi-agent reviewed activity plans with POI recommendations.

### Brand Personality
**Meituan-adjacent hackathon product.** Warm, approachable, trustworthy. Three words: 亲切 (Friendly), 智能 (Smart), 活力 (Energetic).

### Aesthetic Direction
- **Visual tone**: Warm, lifestyle-oriented, clean but not sterile
- **References**: Meituan app (warm yellows), Xiaohongshu (content cards), Dingdong Maicai (fresh, friendly)
- **Anti-references**: Generic AI chat UI (purple/blue gradients, dark glassmorphism), enterprise dashboards, cold tech aesthetic
- **Theme**: Light mode only (warm, inviting). Meituan yellow (#FFC107) as primary accent.

### Design Principles
1. **Progressive disclosure**: Show information in stages matching the agent pipeline (intent → candidates → reviews → best plan → execution)
2. **Warm confidence**: Every element should feel like a knowledgeable friend helping plan, not a machine processing
3. **Scannable hierarchy**: Key information (scores, budget, duration) must be instantly readable; details collapse by default
4. **Alive during wait**: The planning pipeline takes 10-30s; loading state must show meaningful progress and build anticipation
5. **No AI slop**: No glowing borders, no purple gradients, no unnecessary glassmorphism. Clean cards, real shadows, intentional color.

### Technical Constraints
- **Framework**: Streamlit (Python) with heavy CSS customization via `st.markdown` + `unsafe_allow_html`
- **API**: FastAPI backend at localhost:8000, single POST /api/plan endpoint
- **Data shapes**: See backend/schemas/ for Intent, Plan, AgentReview, RoundReviews TypedDicts
- **Existing components**: plan_card, score_matrix, trace_viewer (defined but unused; need rewriting)
- **Browser target**: Modern Chromium (desktop demo for hackathon)
